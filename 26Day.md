### **오늘의 배운 내용**
1. APIDOG : API 개발·테스트·문서화를 한 곳에서 할 수 있는 API 협업 도구
   <br>*ex)```GET, POST, PATCH, DELETE...```*
2. Accept 헤더 : 클라이언트가 받고 싶은 형식의 응답을 서버에 알려주는 HTTP 요청 헤더
   <br>*ex)```application/json, text/html, text/plain, application/xml...```*
3. Content-Type : 지금 이 메시지의 본문(body)가 어떤 형식의 데이터인지 알려주는 HTTP 헤더
   <br>*ex)```application/json, application/xml, application/x-www-form-urlencoded, multipart/form-data```*
4. Cookie : 서버가 브라우저에 저장해 두는 작은 데이터 [HTTP는 stateless(무상태)라서 요청마다 기억을 못 함]
   <br>*ex)```Max-Age/Expires, HttpOnly, Secure, SameSite, Path/Domain...```*
5. Location : 서버가 클라이언트에게 "이 주소 말고 여기로 다시 요청해"라는 헤더
   <br>*ex)```자주 쓰이는 상태 코드: 301, 302, 307, 308```*
6. Cache-Control : 브라우저/프록시/CDN에게 "이 응답을 얼마나, 어떻게 저장할지" 알려주는 헤더
   <br>*ex)```Cache-Control: no-store / Cache-Control: no-cache /```*
   <br>*```Cache-Control: public, max-age=31536000```*
